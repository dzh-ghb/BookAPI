## Цель

Реализовать REST API для управления книгами и авторами.

---
## Ограничения проекта

### Максимальная длительность
- 2-4 учебных дня
- 30-90 минут за занятие

После выполнения требований проект считается законченным (дополнительные улучшения не производить).

---
## Технологии

Обязательно использовать:
- C#
- ASP.NET Core Web API
- Entity Framework Core
- PostgreSQL
- Swagger
---
## Модели

### Author
Поля:
```text
Id
Name
```
---
### Book
Поля:
```text
Id
Title
Year
Pages
AuthorId
```

Связь:
```text
Author (1) ---> (N) Books
```
Один автор может иметь несколько книг.

---
## База данных

Требования:
- Code First
- создание миграций
- применение миграций
- хранение данных в PostgreSQL
---
## API

### Получить всех авторов

```http
GET /api/authors
```
---
### Получить автора по Id

```http
GET /api/authors/{id}
```

Если автор отсутствует:
```http
404 Not Found
```
---
### Создать автора

```http
POST /api/authors
```
---
### Получить все книги

```http
GET /api/books
```
---
### Получить книгу по Id

```http
GET /api/books/{id}
```

Если книга отсутствует:
```http
404 Not Found
```
---
### Создать книгу

```http
POST /api/books
```
---
### Обновить книгу

```http
PUT /api/books/{id}
```

Если книга отсутствует:
```http
404 Not Found
```

---
### Удалить книгу

```http
DELETE /api/books/{id}
```

Если книга отсутствует:
```http
404 Not Found
```

---
### Получить книги автора

```http
GET /api/books/author/{authorId}
```
Возвращает все книги выбранного автора.

---
## Валидация

### Author

Проверять:
```text
Name не пустой
```

---
### Book

Проверять:
```text
Title не пустой
Year > 0
Pages > 0
AuthorId существует
```

При ошибке:
```http
400 Bad Request
```
